# 技术方案

本项目采用“课程版可交付、毕业设计可扩展”的路线。课程版重点展示机械臂动作规划智能体的完整工作流，不直接控制真实设备；后续可在同一规划 JSON 结构上接入乐白 LM3 Python SDK。

## 系统组成

- 前端场景：Three.js 渲染桌面、物体、乐白 LM3 GLB 模型、夹爪动画和末端轨迹。
- 场景图：用结构化 JSON 表示物体 ID、类型、颜色、尺寸、位置、可移动状态，相当于 RGB-D 感知后的简化输入。
- LLM 规划：DashScope/Qwen 根据自然语言任务和 Scene Graph 输出任务级步骤。
- 视觉提示：Three.js 中设置固定手外眼相机，可对当前 3D 场景离屏截图，并按 OpenAI-compatible 多模态格式作为 `image_url` 附加给支持图像输入的千问模型。
- 工具层规划：将任务步骤转换为 `cartesianWaypoints`，再由 LM3 数值 IK 生成 `jointTrajectory`。
- 校验层：检查目标是否存在、是否超出工作空间、是否缺失抓取/释放、IK 是否可达。
- 执行层：Web 仿真中按关节轨迹插值驱动 LM3 模型，夹持物体随 TCP 运动，释放时按几何规则稳定堆叠。

## 当前 JSON 输出

核心规划结果保留三层信息：

```json
{
  "steps": [{ "action": "move_to", "objectId": "red_cube", "pose": { "x": 0.28, "y": 0.35, "z": 0.16 } }],
  "trajectory": [{ "x": 0.28, "y": 0.35, "z": 0.16 }],
  "cartesianWaypoints": [{ "id": "pre_grasp", "position": { "x": 0.28, "y": 0.35, "z": 0.16 }, "gripper": "open" }],
  "jointTrajectory": [{ "waypointId": "pre_grasp", "q": [0, 0, 0, 0, 0, 0], "reachable": true }]
}
```

`steps` 面向用户解释，`trajectory` 用于画末端路径，`jointTrajectory` 用于驱动数字孪生，也为后续 sim-to-real 保留接口。

## 创新点

1. 使用结构化 3D Scene Graph 模拟真实 RGB-D 感知输出，让 LLM 不直接面对像素，而是面对可校验的空间数据。
2. 增加手外眼虚拟相机，把截图作为 VLM 视觉提示，与 Scene Graph 形成“结构化感知 + 图像提示”的双输入展示。
3. 采用 “LLM 任务规划 + 工具化 IK/校验/执行” 的分层架构，避免大模型直接输出不可控关节角。
4. 输出 PlanV2 JSON，将语义步骤、笛卡尔路点、关节轨迹同时保留，便于论文分析和后续真实机械臂接入。
5. 基于乐白 LM3 真实 GLB 模型提取关节层级、关节轴和 TCP 近似偏移，课程版已具备数字孪生雏形。
6. 对课程演示采用确定性几何堆叠，避免轻量物理引擎在方块堆叠时滑落影响展示。

## 局限与后续工作

- 当前 IK 是课程级数值 IK，不等同于工业控制器轨迹规划。
- GLB 关节限位来自公开模型和课程假设，真实设备控制前必须以乐白官方参数和现场标定为准。
- 当前没有相机标定、手眼标定、碰撞体精确建模和急停策略。
- 真实机器人版本建议新增 FastAPI/Python 驱动层，只接受通过校验的 dry-run 轨迹，再调用 `lebai_sdk`。
