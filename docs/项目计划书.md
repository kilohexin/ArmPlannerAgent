# ArmPlannerAgent 项目技术栈与实现方法总结

------

## 一、项目核心定位

**项目名称**：ArmPlannerAgent（机械臂规划智能体）

**选题方向**：机械臂 / 机器人的动作自动规划

**项目属性**：面向机器人抓取任务的专业级智能体应用， AI4S 课程课题要求。

**核心目标**：以大语言模型为决策核心，实现自然语言指令→机械臂运动规划→Web 端物理仿真抓取的端到端自动化工作流，支持方块抓取、堆叠等典型工业场景仿真验证。

------

## 二、完整技术栈（分层架构）

### 1. 3D 渲染与交互层

- **核心引擎**：Three.js —— Web 端 3D 渲染标准方案，负责机械臂、夹爪、场景物体的可视化渲染，支持 PBR 材质、阴影、轨道交互
- **交互控制**：OrbitControls —— 提供场景视角的缩放、旋转、平移，便于观察抓取过程
- **UI 交互**：原生 HTML/CSS/JavaScript —— 实现指令输入框、关节滑块、抓取状态展示等轻量交互控件

### 2. 机械臂与模型层

- **机械臂本体**：Universal Robots UR5 官方标准 URDF 模型（源自 ROS Industrial 社区 `fmauch/universal_robot` 仓库），包含完整 DH 参数、关节限位、连杆尺寸
- **末端夹爪**：Robotiq 2F-85 工业二指夹爪 URDF 模型（源自 `ros-industrial/robotiq` 仓库），标准法兰接口与 UR5 完美适配，支持单关节联动控制
- **模型加载器**：`urdf-loader`（Three.js 生态官方插件）—— 直接解析 URDF 文件，自动构建关节层级树，保留完整的关节名称、限位、旋转轴信息
- **坐标系适配**：URDF 原生 Z-up 坐标系通过根节点旋转变换对齐 Three.js Y-up 坐标系，保证运动学参数与官方标准完全一致

### 3. 物理仿真层（方案 B：真实物理抓取）

- **物理引擎**：Cannon-es —— 纯 JavaScript 实现的轻量 3D 物理引擎，Web 端兼容性最佳，支持刚体碰撞、摩擦、重力、约束
- 刚体分类设计：
  - 地面：静态刚体（mass=0），提供碰撞支撑
  - 机械臂与夹爪：运动学刚体（Kinematic），由关节角度直接控制位姿，不受外力影响，可正确碰撞交互
  - 抓取物体（方块）：动力学刚体（Dynamic），受重力、摩擦力作用，可被夹爪夹持、堆叠、掉落
- **抓取实现**：基于接触摩擦力的物理抓取方案，夹爪闭合时双指挤压方块，通过摩擦力实现稳定夹持；张开时接触消失，方块自然下落
- **优化策略**：使用基础几何体（盒体）简化碰撞体，避免高精度网格带来的性能损耗；通过摩擦系数、接触偏移参数调优保证抓取稳定性

### 4. 大模型智能体层

- **核心大模型**：阿里云通义千问 `qwen3.7-plus`（课程指定）—— 负责自然语言指令理解、任务拆解、运动规划决策
- **调用方式**：前端直连通义千问 OpenAPI，通过 HTTP 请求实现指令交互
- **知识库内置**：UR5 机械臂 DH 参数、关节限位范围、夹爪行程参数、抓取运动规范等专业领域知识
- **工具集**：逆运动学求解函数、关节角度控制接口、物理仿真启停接口、场景物体坐标查询接口

### 5. 开发辅助工具

- **开发环境**：VS Code
- **AI 编程辅助**：OpenAI Codex 官方 VS Code 插件（课程指定 codex 工具）—— 全程辅助代码生成、架构设计、bug 排查、功能重构，符合课程「全程 AI 指导完成」的要求
- **配套效率插件**：Live Server（本地实时预览）、Error Lens（行内报错提示）、GLTF Viewer（模型预览）

### 6. 部署层

- **部署平台**：GitHub Pages（课程指定免费方案）
- **部署形态**：纯静态前端应用，无后端服务依赖，单页应用架构，可直接生成公开可访问的 URL

## 三、核心实现方法与工作流

### 1. 机械臂与夹爪控制实现

1. 通过 `urdf-loader` 加载 UR5 与 Robotiq 夹爪的 URDF 文件，配置 `package` 路径映射解析网格资源
2. 根节点执行 `rotation.x = -Math.PI / 2` 完成坐标系转换，保证单位、尺寸与官方参数完全一致
3. 封装统一关节控制接口，通过 `setJointValue()` 精准控制 6 个旋转关节与夹爪移动关节，支持关节限位保护与平滑缓动动画
4. 夹爪通过 mimic 联动机制实现单关节控制双指同步开合，控制逻辑与真机完全对齐

### 2. 物理抓取与堆叠实现

1. 初始化 Cannon-es 物理世界，设置重力与仿真步长，创建地面静态刚体
2. 为机械臂各连杆、夹爪手指创建对应简化碰撞体，每帧同步渲染模型的位姿到物理刚体
3. 创建可抓取方块动力学刚体，物理引擎自动计算重力、碰撞、摩擦效果
4. 夹爪闭合时，双指碰撞体与方块产生接触约束，依靠摩擦力实现稳定夹持；夹爪张开时接触解除，方块受重力自然下落，支持多层物理堆叠
5. 渲染循环中每帧执行物理步进，并将物理刚体位姿同步回 Three.js 渲染模型，实现物理与视觉的统一

### 3. 大模型智能体工作流（对应课程「LLM + 工具 + 知识库 + 工作流」框架）

1. **指令输入**：用户输入自然语言任务（如「将红色方块堆叠到蓝色方块上方」）
2. **LLM 解析与规划**：通义千问结合内置机械臂知识库，拆解任务为分步动作序列（移动到方块上方→下降→夹爪闭合→抬升→移动到目标位置→下降→夹爪张开）
3. **工具调用**：依次调用逆运动学工具求解对应关节角度，调用机械臂控制接口执行运动，调用夹爪接口执行开合
4. **仿真执行**：物理引擎同步运行，实时反馈抓取状态与物体位置
5. **结果反馈**：任务完成后向用户返回执行结果与状态信息
