# 任务模式与夹爪校准设计

## 目标

在现有 pick-place 演示基础上，增加两类交互任务：

- `pick_only`：只抓起目标物体并保持夹持，例如“夹起红色方块”。
- `relative_place`：把目标物体放到当前位置的相对方向，例如“把红色方块往右放一点”。

同时把夹爪 TCP 校准从源码常量改为运行时配置，减少手动改代码造成的坐标偏移和 IK 误判。

## 状态规则

- 未夹持物体时：
  - `把 A 放到 B 上` 生成完整抓取、移动、释放流程。
  - `夹起 A` 生成抓取和抬升流程，最后保持夹持状态。
  - `把 A 往右放一点` 先抓取 A，再移动到相对目标点并释放。
- 已夹持物体时：
  - `把它/当前物体放到 B 上` 直接移动并释放，不重复抓取。
  - `把它往右放一点` 以当前夹持物为目标，移动到相对目标点并释放。
  - `夹起 B` 如果 B 不是当前夹持物，拒绝执行，提示先放下当前物体。

## 规划结构

`ManipulationPlan` 增加 `taskType`：

- `pick_place`
- `pick_only`
- `relative_place`

校验器按任务类型判断必要动作：

- `pick_place` 和未夹持的 `relative_place` 需要 `grasp + release`。
- `pick_only` 需要 `grasp`，不要求 `release`。
- 已夹持的放置类任务只需要移动和 `release`。

## 夹爪校准

新增运行时 TCP 校准配置：

- 默认值来自代码。
- 浏览器本地存储保存用户校准值。
- 规划器和执行器统一读取同一组校准值，避免计划与执行坐标不一致。

第一版只提供数值输入和保存，不引入复杂自动标定。后续如果要更真实，可加入“夹爪中心可视化标记”和“拖动标定点”。

## 测试

新增测试覆盖：

- 未夹持时 `夹起红色方块` 生成 `pick_only`，不被缺少 release 拦截。
- 夹持红色方块后 `把它往右放一点` 生成 `relative_place`，目标位置 x 增加。
- 夹持红色方块时 `夹起蓝色方块` 被拒绝。
- 严重 IK 残差仍被拦截，轻微残差允许执行。
