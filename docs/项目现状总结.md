# ArmPlannerAgent 项目现状总结

> 撰写日期：2026-07-04  
> 用途：论文参考

---

## 一、项目定位

ArmPlannerAgent 是一个面向机械臂抓取操作的智能体原型系统。核心目标：**以自然语言为输入，通过大语言模型进行任务理解与规划，在浏览器端 3D 仿真环境中完成抓取与放置操作的自动化执行。**

目标平台：乐白 LM3 六轴协作机械臂（课程版使用 GLB 数字模型进行 Web 端仿真，不控制真实硬件）。

### 架构特点

- **纯前端**：无后端服务依赖，TypeScript + Vite + Three.js 单页应用
- **LLM 优先**：规划以 LLM 为主，本地规则为兜底，LLM 不可用时系统自动降级
- **安全边界**：LLM 输出经过确定性代码校验后才可执行，校验不通过拒绝执行

| 层次 | 技术 | 用途 |
|------|------|------|

| 前端构建 | Vite + TypeScript | 开发/构建/静态部署 |
| 测试 | Vitest | 单元测试 |

## 四、智能体工作流

### 4.1 完整管线

```
用户自然语言指令
       │
       ▼
┌─ ① 场景感知 ────────────────────────────────────────────┐
│  采集结构化场景图 (Scene Graph)：                          │
│  - 所有物体的 id、标签、类型、颜色、位置、尺寸、是否可移动  │
│  - 机器人状态（基座位置、工作半径、当前夹持物、关节角度）    │
│  - 执行历史（最近 5 条记录）                              │
│  - 可选：手外眼相机截图（512×384 JPEG，作为 VLM 视觉提示） │
└──────────────────────────────────────────────────────────┘
       │
       ▼
┌─ ② LLM 任务规划 ────────────────────────────────────────┐
│  System Prompt 约束：                                     │
│  - LM3 工作半径 0.85m，桌面高度 0.035m                     │
│  - 安全移动高度 0.35m                                     │
│  - 从上方接近物体（top-grasp）                            │
│  - 堆叠时计算叠加高度                                     │
│                                                          │
│  LLM 输出 JSON: { steps[], confidence, explanation }     │
│  每个 step: { action, objectId, targetPosition,           │
│              gripperAction, description }                 │
│                                                          │
│  模型: Qwen3.5-Flash (默认)，支持切换                      │
│  API: OpenAI 兼容接口，用户自填 Key                        │
└──────────────────────────────────────────────────────────┘
       │
       ▼
┌─ ③ 确定性安全校验 ───────────────────────────────────────┐
│  全部由代码实现，不依赖 LLM：                              │
│  ✅ 计划非空                                              │
│  ✅ 所有引用的物体在场景中存在                              │
│  ✅ 所有目标坐标在 0.85m 工作半径内（XZ 距离计算）          │
│  ✅ 抓取目标标记为可移动                                   │
│  ✅ 所有高度坐标不低于桌面 (y ≥ 0.01m)                     │
│  ✅ 步数不超过上限 (≤ 30)                                  │
│  ✅ IK 关节误差在可接受范围 (< 0.09m)                      │
│  ✅ 抓取/释放动作顺序正确                                  │
└──────────────────────────────────────────────────────────┘
       │
       ├── 不通过 → 返回修复提示，不执行
       │
       └── 通过
              │
              ▼
┌─ ④ 运动规划与执行 ───────────────────────────────────────┐
│  - 将 LLM 步骤转为笛卡尔航点                               │
│  - 逆运动学求解：笛卡尔位姿 → 6 关节角度                   │
│    · CCD-like 数值迭代                                   │
│    · 腕部对齐时启用多起点搜索 (9 个种子)                    │
│  - 关节空间线性插值 (40 帧, ~16ms/帧)                      │
│  - 夹爪动画控制 (AnimationMixer, 0-1 映射)                │
│  - 物理引擎同步：被抓物体跟随末端运动                       │
└──────────────────────────────────────────────────────────┘
```

### 4.2 LLM 降级策略

| 条件 | 行为 |
|------|------|
| 无 API Key | 直接使用本地规则规划 |
| LLM 调用异常/超时 | 退回本地规则规划 |
| LLM 置信度 < 0.4 | 退回本地规则规划 |
| LLM 通过 + confidence ≥ 0.4 | 使用 LLM 规划 |

### 4.3 本地规则规划（兜底）

基于关键词匹配的任务类型检测：

| 任务类型 | 触发关键词 | 说明 |
|----------|-----------|------|
| `pick_place` | 默认 | 抓取+放置，8 步 |
| `pick_only` | 抓起/拿起/pick up/grab/grasp | 仅抓取，4 步 |
| `relative_place` | 左边/右边/前面/后面/left/right | 相对位置放置 |
| 手中已有物体 | 自动检测 `robot.holding` | 仅执行放置 |

## 五、3D 仿真环境

### 5.1 场景组成

- **桌面**：Box 网格 (1.25×0.035×0.88m)，静态碰撞体
- **初始物体**：红/蓝/绿方块 + 黄/紫圆柱体（共 5 个），动态碰撞体 (mass=0.3kg)
- **机械臂**：乐白 LM3 GLB 模型，含 6 旋转关节 + 夹爪动画
- **光源**：半球光 + 方向光（带阴影）
- **固定相机**：手外眼相机，模拟外部 RGB-D 传感器视角

### 5.2 交互功能

| 功能 | 实现 |
|------|------|
| 视角切换 | 俯视/正视/等轴/手外眼相机视角 |
| 物体添加 | 添加方块/圆柱体（随机位置） |
| 物体拖拽 | 鼠标拖拽物体在桌面平面移动 |
| 场景持久化 | localStorage 存取完整场景状态 |
| 手外眼截图 | 512×384 JPEG，作为 VLM 多模态输入 |

## 六、运动学系统

### 6.1 正运动学 (FK)

基于 GLB 模型关节层级链的四元数累积计算，给定 6 个关节角度，输出 TCP 世界位姿和工具方向向量。

### 6.2 逆运动学 (IK)

自研 CCD-like 数值求解器：

| 参数 | 值 |
|------|----|
| 关节数 | 6 |
| 最大迭代 | 120 |
| 收敛容差 | 0.018m |
| 多起点 | 9 个种子配置（腕部对齐时启用） |
| 关节限位 | ±π |
| 末端方向对齐 | 支持，通过约束 J4-J6 实现 |

## 七、物理仿真

- **引擎**：Cannon-es
- **重力**：-9.81 m/s²
- **桌面**：静态刚体 (mass=0)
- **物体**：动态刚体 (mass=0.3kg)，Box/Cylinder 碰撞体
- **夹持**：被抓物体切换为运动学模式，每帧跟随 TCP 位姿同步
- **释放**：恢复动态刚体，自然受重力下落
- **每帧同步**：physics body → Three.js mesh → sceneState

## 八、视觉感知模拟

系统通过一个固定位置的手外眼（eye-to-hand）相机模拟外部视觉传感器：

| 参数 | 值 |
|------|----|
| 位置 | (1.18, 0.90, 0.90) |
| 目标 | (0.24, 0.18, 0.02) |
| FOV | 68° |
| 输出 | 512×384 JPEG (quality 0.86) |
| 用途 | 可选附加到 LLM 请求中，作为 VLM 视觉提示 |

## 十一、关键设计决策

1. **LLM 全权规划 + 确定性校验**：LLM 输出完整执行序列（动作+坐标），确定性代码负责安全把关。LLM 不直接控制机械臂。

2. **纯前端架构**：无后端服务，API Key 由用户自行填入并存于浏览器 localStorage。系统在无 Key 时自动降级为本地规则规划，保证可用性。

3. **GLB 模型驱动**：使用乐白 LM3 的 GLB 模型文件，通过 Three.js AnimationMixer 控制夹爪动画。逆运动学基于模型的关节层级链实现。

4. **多模态 LLM 输入**：场景以结构化 JSON + 可选相机截图的组合发送给 LLM，支持纯文本和图文混合两种模式。

5. **物理+运动学混合**：机械臂运动由运动学驱动（关节插值），场景物体由物理引擎驱动（重力/碰撞）。夹持时物体在两者之间切换。
