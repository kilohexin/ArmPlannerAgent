# 接口说明

当前课程版没有后端接口，主接口是前端内部的规划 JSON。后续加入 FastAPI 后端时，建议沿用以下数据结构，避免重写前端和论文分析部分。

## PlanResponse

```ts
interface PlanResponse {
  plan: ManipulationPlan;
  validation: ValidationResult;
  notes: string[];
}
```

## ManipulationPlan 关键字段

- `steps`：面向用户和日志的语义步骤。
- `trajectory`：末端执行器笛卡尔轨迹点，用于场景中画线。
- `cartesianWaypoints`：带夹爪状态、速度、融合半径的任务级路点。
- `jointTrajectory`：LM3 六关节角序列，字段 `q` 可作为真实 SDK 前的候选输入。
- `robotModel`：当前机器人模型引用，课程版为 `lebai-lm3`。
- `executionMode`：当前为 `sim`，真实机器人扩展时可加入 `real`。

## 后续 FastAPI 建议

- `POST /api/plan`：输入任务和 Scene Graph，返回 `PlanResponse`。
- `POST /api/validate`：输入 `ManipulationPlan`，返回校验结果。
- `POST /api/dry-run`：只做真实设备安全校验，不驱动机械臂。
- `POST /api/execute`：仅在 dry-run 通过、人工确认后调用乐白 SDK。
