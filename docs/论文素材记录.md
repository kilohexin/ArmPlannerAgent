# 论文素材记录

## 可写入论文的工程事实

- 系统没有让大模型直接输出关节角，而是采用“LLM 语义规划 + 工具层校验 + IK 转换 + Web 仿真执行”的分层设计。
- Web 场景图模拟真实 RGB-D 感知之后的结构化空间信息，包含物体类别、颜色、位置、尺寸和可移动状态。
- 规划结果同时保存语义步骤、末端笛卡尔路点和 LM3 六关节轨迹，便于解释智能体如何从自然语言过渡到可执行动作。
- 系统增加了手外眼虚拟相机，能够把 3D 场景截图作为可选视觉提示传入多模态模型，用来展示真实 RGB-D/VLM 输入的流程雏形。
- 真实设备路线围绕实验室乐白 LM3 和 LMG-90 夹爪，不再依赖泛化的开源 UR5 模型。
- 课程版默认禁止真实机器人执行，体现安全边界；后续真实执行需要 dry-run、人工确认和官方 SDK 驱动。

## 可讨论的问题

- STP/CAD 模型不能直接用于 Web 关节动画，需要转为 GLB，并且最好按关节拆分或保留可旋转节点。
- 真实机器人落地需要相机标定、手眼标定、坐标系转换、碰撞体建模和速度/加速度限制。
- 当前使用几何堆叠而不是完全物理仿真，是为了保证课程演示稳定；论文中应解释这是工程取舍。
- 当前 IK 是课程级数值算法，后续可替换为更严格的解析 IK、MoveIt 风格规划或乐白控制器侧轨迹规划。
- 展望部分可以写：把当前虚拟相机替换为真实 RGB-D 相机，把 Scene Graph 从人工维护改为视觉检测生成，再用乐白 SDK 完成 sim-to-real。
